# Уровень L1

## Назначение

`L1` — это нижний уровень работы с железом.

На этом уровне находятся только:

- исполнение команд на моторы;
- получение данных с датчиков;
- простое управление исполнительными устройствами.

`L1` не рассчитывает движение и не строит состояние машинки. Он только:

- принимает готовые команды для железа;
- применяет их;
- возвращает измеренные данные.

## Что входит в L1

В текущей реализации в `L1` входят:

- модели данных уровня;
- чистый сервис уровня;
- моторы;
- IMU;
- ультразвуковой датчик;
- сервопривод головы.

Файлы:

- `src/application/services/l1_models.py`
- `src/application/services/l1_service.py`
- `src/infrastructures/motor.py`
- `src/infrastructures/imu.py`
- `src/infrastructures/ultrasonic.py`
- `src/infrastructures/head_servo.py`

## Что происходит в L1

### Формулы уровня L1

На уровне `L1` используются только те соотношения, которые нужны для передачи команд на устройства
и выдачи измеренных данных.

#### 1. Команды бортов

В `L1` на моторы передаются команды:

```text
u_L in [-100, 100]
u_R in [-100, 100]
```

где:

- `u_L` — команда левого борта;
- `u_R` — команда правого борта.

После этого каждая команда ограничивается допустимым диапазоном:

```text
если u_L > 100, то u_L = 100
если u_L < -100, то u_L = -100

если u_R > 100, то u_R = 100
если u_R < -100, то u_R = -100
```

#### 2. Переход к управляющему значению драйвера

Внутри моторного слоя команда в процентах переводится во внутреннее значение драйвера:

```text
t_L = u_L / 100
t_R = u_R / 100
```

где:

- `t_L`, `t_R` — внутренние значения для управления мотором;
- далее они ещё умножаются на знак физической установки моторов.

#### 3. Угловая скорость по гироскопу

В новом изолированном контуре `L1` наружу отдаёт измеренную угловую скорость вокруг вертикальной
оси. Внутри драйвера IMU используется скорость, полученная после вычитания смещения гироскопа:

```text
omega_z = s_g * (g_z - b_z) / k_g
```

где:

- `g_z` — сырое значение гироскопа;
- `b_z` — смещение, найденное при калибровке;
- `k_g` — коэффициент перевода в скорость;
- `s_g` — знак, учитывающий ориентацию платы.

На уровне `L1` эта величина только читается и передаётся наружу.

#### 4. Ускорение по осям

Ускорения акселерометра в `L1` переводятся из сырых значений в физические единицы:

```text
a_x = s_x * r_x / k_a * g
a_y = s_y * r_y / k_a * g
a_z = s_z * r_z / k_a * g
```

где:

- `r_x, r_y, r_z` — сырые значения акселерометра;
- `k_a` — коэффициент перевода;
- `g` — ускорение свободного падения;
- `s_x, s_y, s_z` — знаки осей с учётом ориентации платы.

### Управление моторами

Основная задача `L1` для движения — независимо задать команды левой и правой гусеницы.

Рабочие входные значения:

- `u_left`
- `u_right`

Смысл:

- это команды для левого и правого борта;
- единицы представления — проценты со знаком;
- рабочий диапазон: `[-100; 100]`.

Сейчас это реализовано методом:

- `src/infrastructures/motor.py`
  `set_tracks(left_speed_percent, right_speed_percent)`

Что делает этот метод:

- принимает команду для левого и правого борта;
- ограничивает каждое значение диапазоном `[-100; 100]`;
- учитывает направление установки моторов;
- переводит команду в управляющее значение драйвера мотора по формулам выше;
- передаёт её на физические моторы.

Также в моторном слое сохранены совместимые команды более высокого уровня:

- движение вперёд;
- движение назад;
- поворот на месте;
- остановка.

Они нужны для текущей совместимости, но базовый интерфейс `L1` для движения — именно независимое
управление двумя бортами.

### Чистый сервис уровня

Для нового изолированного контура используется отдельный чистый сервис `L1`.

Файлы:

- `src/application/services/l1_models.py`
- `src/application/services/l1_service.py`

Этот сервис:

- принимает команду левого и правого борта;
- сразу передаёт её в моторы;
- читает угловую скорость, ускорения и расстояние;
- не вычисляет угол положения и не строит состояние машинки.

Основные модели данных:

- `L1TrackCommand`
- `L1SensorState`

Основные действия:

- `apply_track_command(...)`
- `read_sensors()`
- `stop_motion()`
- `start_imu(...)`
- `stop_imu()`
- `set_head_angle(...)`

### Чтение IMU

`L1` читает IMU и хранит текущие измеренные значения.

Сейчас для нового контура используются:

- угловая скорость вокруг вертикальной оси;
- ускорения по трём осям.

Методы:

- `src/infrastructures/imu.py`
  `get_angular_speed_z_deg_per_sec()`
- `src/infrastructures/imu.py`
  `get_acceleration_xyz_m_s2()`

Что происходит внутри:

- открывается I2C-шина;
- MPU6050 выводится из сна;
- выполняется калибровка нулевого смещения гироскопа;
- в фоновом цикле читаются сырые данные;
- вычисляется угловая скорость по оси `Z` по формуле выше;
- ускорения переводятся в метры на секунду в квадрате по формулам выше.

Важно:

- данные акселерометра уже читаются;
- они доступны наружу;
- в самом `L1` они не превращаются в координаты, скорость или угол машинки нового контура.

### Чтение ультразвукового датчика

`L1` умеет измерять расстояние до препятствия.

Метод:

- `src/infrastructures/ultrasonic.py`
  `measure_distance_cm()`

Что происходит внутри:

- при первом обращении инициализируется датчик;
- читается текущее расстояние;
- расстояние возвращается в сантиметрах;
- при ошибке возвращается запасное безопасное значение.

### Управление сервоприводом головы

`L1` умеет задавать угол головы или датчика.

Основной метод:

- `src/infrastructures/head_servo.py`
  `set_angle(angle_deg)`

Что происходит внутри:

- при первом обращении инициализируется PCA9685;
- значение угла ограничивается рабочим диапазоном;
- угол передаётся на сервопривод.

## Какие данные принимает L1

Если брать только минимальный интерфейс для движения, `L1` принимает:

- `u_left`
- `u_right`

Описание:

- `u_left` — команда левой гусенице;
- `u_right` — команда правой гусенице;
- обе величины передаются в процентах со знаком;
- допустимый диапазон каждой величины: `[-100; 100]`.

Если брать дополнительные устройства, `L1` также может принимать:

- `angle_deg` для сервопривода головы.

## Какие данные выдаёт L1

`L1` выдаёт только измеренные или непосредственно считанные данные.

Сейчас наружу доступны:

- `angular_speed_z` — текущая угловая скорость вокруг вертикальной оси;
- `accel_x` — текущее ускорение по продольной оси;
- `accel_y` — текущее ускорение по поперечной оси;
- `accel_z` — текущее ускорение по вертикальной оси;
- `distance_cm` — расстояние до препятствия.

То есть `L1` выдаёт:

- данные IMU без расчёта положения;
- данные ультразвукового датчика.

## Граница L1

Граница `L1` должна оставаться простой.

Внутрь `L1` входят:

- команды на моторы;
- команды на сервопривод.

Из `L1` выходят:

- данные IMU;
- расстояние с ультразвукового датчика.

`L1` не должен принимать:

- координаты цели;
- маршрут;
- желаемую скорость корпуса;
- ошибку движения;
- параметры траектории.

`L1` не должен выдавать:

- положение машинки на плоскости;
- оценку скорости машинки;
- оценку полного состояния;
- команды следующего шага движения.

## Что L1 делает сам, а что нет

`L1` делает сам:

- ограничение команд допустимым диапазоном;
- учёт направления установки моторов;
- перевод команд бортов в сигналы для драйвера;
- чтение и калибровку IMU;
- чтение расстояния;
- установку угла сервопривода.

`L1` не делает:

- расчёт кинематики;
- преобразование движения корпуса в движение бортов;
- интегрирование пути по плоскости;
- выбор направления;
- расчёт ошибки до цели;
- принятие решений по движению.
