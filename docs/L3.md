# Уровень L3

## Назначение

`L3` — это верхний уровень нового изолированного контура движения.

Именно на этом уровне:
- задаётся одна целевая точка или маршрут;
- строится маршрут с обходом заранее известных препятствий;
- при внезапном обнаружении препятствия перестраивается остаток маршрута;
- выбирается активная точка движения;
- считается ошибка по расстоянию и направлению;
- формируются желаемые `v_des` и `omega_des`;
- выполняется плавное торможение перед препятствием;
- принимается решение о переходе к следующей точке, завершении маршрута или остановке.

`L3` не работает с железом напрямую и не переводит `v_des`, `omega_des` в команды бортов. Это делает `L2`.

## Что входит в L3

В текущей реализации в `L3` входят:
- модели данных уровня;
- контроллер движения к активной точке;
- планировщик маршрута с обходом заранее известных препятствий;
- логика перестроения маршрута для впервые обнаруженного препятствия;
- единый сервис верхнего уровня.

Файлы:
- `src/application/services/l3_models.py`
- `src/application/services/goal_point_controller.py`
- `src/application/services/path_planner.py`
- `src/application/services/l3_service.py`
- `src/application/factories.py`
- `src/config/settings.py`

## Модели данных уровня

В `L3` используются:
- `TargetPoint` — одна целевая точка;
- `TargetRoute` — исходный маршрут как последовательность целевых точек;
- `KnownObstacle` — заранее известное препятствие;
- `PlannedRoute` — результат планирования маршрута;
- `L3Command` — желаемая линейная и угловая скорость корпуса;
- `GoalTrackingCommand` — результат расчёта движения к активной точке;
- `L3State` — текущее состояние верхнего уровня.

## Какие величины используются в L3

Для расчётов используются:
- `x_goal`, `y_goal` — координаты активной точки;
- `x`, `y` — текущие координаты машинки;
- `theta` — текущий угол машинки;
- `dx`, `dy` — ошибка по координатам;
- `rho` — расстояние до активной точки;
- `theta_target` — направление на активную точку;
- `alpha` — ошибка по углу;
- `v_des` — желаемая линейная скорость корпуса;
- `omega_des` — желаемая угловая скорость корпуса;
- `distance_cm` — расстояние до препятствия по ультразвуку.

Единицы измерения:
- координаты — сантиметры;
- углы — градусы;
- линейная скорость — `см/с`;
- угловая скорость — `град/с`.

## Полная цепочка работы L3

### 1. Принимается цель или маршрут

На вход `L3` подаётся либо:

```text
target = (x_goal, y_goal)
```

либо:

```text
route = [(x_1, y_1), (x_2, y_2), ..., (x_n, y_n)]
```

### 2. Планируется маршрут с учётом известных препятствий

Если передан список известных препятствий, `L3` запускает планировщик:
- проверяет каждый отрезок маршрута;
- ищет пересечение с препятствием;
- при необходимости вставляет обходные точки;
- если обход в допустимом коридоре невозможен, маршрут помечается как `impossible`.

На текущий момент планировщик:
- использует препятствия в виде кругов `KnownObstacle`;
- добавляет двухточечный обход с одной из сторон;
- выбирает более короткий допустимый вариант;
- ограничивает боковое смещение обхода.

Если препятствие заранее не было известно, но было впервые обнаружено датчиком
по ходу движения, `L3`:
- оценивает его координаты по текущей позе и измеренной дальности;
- добавляет его во внутренний список временных препятствий;
- перестраивает путь от текущей позиции до оставшихся целевых точек;
- после обхода возвращается на исходную траекторию через оставшиеся точки маршрута.

### 3. Выбирается активная точка маршрута

Если маршрут успешно построен, `L3` хранит:
- весь список точек;
- индекс активной точки;
- общее количество точек.

### 4. Из `L2` берётся текущее состояние машинки

Из `L2` используются:
- `x`
- `y`
- `theta`
- `distance_cm`

`L3` не строит состояние корпуса сам.

### 5. Считается ошибка по координатам

```text
dx = x_goal - x
dy = y_goal - y
rho = sqrt(dx^2 + dy^2)
theta_target = atan2(dy, dx)
alpha = theta_target - theta
```

После этого `alpha` нормализуется в диапазон `[-180; 180]`.

### 6. Проверяется достижение активной точки

Если:

```text
rho <= position_tolerance
```

То точка считается достигнутой.

Дальше:
- если впереди есть следующая точка, `L3` переключается на неё;
- если следующей точки нет, маршрут завершается.

### 7. Проверяется расстояние до препятствия

Если:

```text
distance_cm <= obstacle_stop_distance_cm
```

То движение останавливается:

```text
v_des = 0
omega_des = 0
status = blocked
```

После остановки `L3` пробует перестроить остаток маршрута.

Координаты неизвестного препятствия оцениваются так:

```text
obstacle_x = x + cos(theta) * (distance_cm + unknown_obstacle_radius_cm)
obstacle_y = y + sin(theta) * (distance_cm + unknown_obstacle_radius_cm)
```

Дальше это препятствие добавляется в планировщик как временно известное.

Если перестроение удалось:

```text
status = tracking
planner_status = replanned_dynamic
```

Если перестроение не удалось:

```text
status = unreachable
planner_status = impossible
```

### 8. Считается желаемая линейная скорость

Поступательная скорость строится пропорционально расстоянию до активной точки:

```text
v_des = linear_speed_gain * rho
```

После этого она ограничивается сверху:

```text
v_des <= max_linear_speed
```

### 9. Линейная скорость уменьшается при большой ошибке по курсу

Чтобы машинка не уезжала далеко вперёд при большом отклонении по направлению:

```text
v_des = v_des * max(0, cos(alpha))
```

Это даёт естественное замедление поступательного движения, когда машинка сильно отвернута от цели.

### 10. Выполняется плавное торможение перед препятствием

Если расстояние до препятствия попадает в зону замедления:

```text
obstacle_stop_distance_cm < distance_cm < obstacle_slowdown_distance_cm
```

То поступательная скорость масштабируется:

```text
speed_ratio = (distance_cm - obstacle_stop_distance_cm)
              / (obstacle_slowdown_distance_cm - obstacle_stop_distance_cm)

v_des = v_des * speed_ratio
```

Это значит:
- далеко от препятствия скорость не режется;
- ближе к препятствию она плавно уменьшается;
- на границе остановки становится нулевой.

### 11. Считается желаемая угловая скорость

```text
omega_des = angular_speed_gain * alpha
```

После этого:

```text
-max_angular_speed <= omega_des <= max_angular_speed
```

### 12. `L3` передаёт желаемое движение в `L2`

На выходе `L3` формирует:

```text
v_des
omega_des
```

Именно эти значения передаются в `L2`, а уже `L2` переводит их в команды бортов.

## Где реализована каждая часть

### Контроллер движения к точке

Файл:
- `src/application/services/goal_point_controller.py`

Основной объект:
- `GoalPointController`

Что делает:
- считает ошибку по расстоянию и углу;
- строит `v_des` и `omega_des`;
- включает плавное торможение перед препятствием;
- определяет состояния `tracking`, `reached`, `blocked`.

### Планировщик пути

Файл:
- `src/application/services/path_planner.py`

Основной объект:
- `PathPlanner`

Что делает:
- строит маршрут от текущей позиции до цели или набора точек;
- вставляет обходные точки вокруг заранее известных препятствий;
- объявляет цель недостижимой, если обход не помещается в допустимый коридор.

### Единый сервис уровня

Файл:
- `src/application/services/l3_service.py`

Основной объект:
- `L3Service`

Что делает:
- принимает цель или маршрут;
- запускает планировщик;
- хранит активную точку;
- вызывает контроллер цели;
- передаёт `v_des`, `omega_des` в `L2`;
- переключает маршрут на следующую точку;
- хранит состояние уровня.

## Какие данные принимает L3

`L3` принимает:
- целевую точку или маршрут;
- список заранее известных препятствий;
- текущее состояние из `L2`;
- расстояние до препятствия из состояния `L2`.

## Какие данные выдаёт L3

`L3` выдаёт:
- `linear_speed_cm_per_sec`
- `angular_speed_deg_per_sec`
- статус уровня;
- режим `idle`, `point`, `route`;
- статус планировщика;
- индекс активной точки;
- число точек в маршруте;
- текущую ошибку по расстоянию;
- текущую ошибку по углу;
- целевое направление на активную точку.
- координаты последнего обнаруженного неизвестного препятствия;
- радиус последнего обнаруженного неизвестного препятствия;
- тип последнего обнаруженного препятствия.

## Какие параметры хранятся в L3

Для расчётов `L3` использует:
- `l3_position_tolerance_cm`
- `l3_linear_speed_gain`
- `l3_angular_speed_gain`
- `l3_max_linear_speed_cm_per_sec`
- `l3_max_angular_speed_deg_per_sec`
- `l3_obstacle_stop_distance_cm`
- `l3_obstacle_slowdown_distance_cm`
- `l3_planner_obstacle_clearance_cm`
- `l3_planner_max_detour_offset_cm`
- `l3_planner_max_waypoints`
- `l3_unknown_obstacle_radius_cm`

## Что нужно подобрать в L3

Для практической настройки поведения уровня желательно подобрать:
- допуск достижения точки;
- коэффициенты поступательной и угловой скорости;
- максимальные скорости;
- расстояние начала замедления;
- расстояние полной остановки;
- минимальный зазор при обходе;
- максимально допустимое боковое смещение обхода.
- оценочный радиус впервые обнаруженного препятствия.

## Что L3 делает сам

`L3` сам выполняет:
- хранение цели и маршрута;
- планирование маршрута вокруг известных препятствий;
- перестроение маршрута при неожиданно обнаруженном препятствии;
- выбор активной точки;
- расчёт ошибки по положению;
- расчёт ошибки по направлению;
- плавное торможение перед препятствием;
- переход к следующей точке;
- остановку при достижении последней точки;
- остановку при препятствии;
- объявление цели недостижимой, если обход невозможен.

## Что L3 пока не делает

`L3` пока не делает:
- прямую работу с железом;
- перевод `v_des`, `omega_des` в команды бортов;
- оценку `x`, `y`, `theta` из датчиков;
- перестройку пути по карте окружения в реальном времени;
- сложные кривые траектории с ограничениями на кривизну.

## Связь с целями команды

Текущая реализация `L3` уже закрывает основу трёх целевых сценариев:

- движение по маршруту:
  маршрут из точек поддерживается;
- плавное торможение перед препятствием:
  реализовано через `obstacle_slowdown_distance_cm` и `obstacle_stop_distance_cm`;
- объезд заранее известных препятствий:
  реализован через `PathPlanner`;
- объезд заранее неизвестных препятствий:
  реализован через перестроение остатка маршрута после первого обнаружения;
- достижение выбранной пользователем точки:
  реализовано как частный случай маршрута из одной точки.

Что ещё остаётся уже не на уровне архитектуры, а на уровне доводки на железе:
- подбор коэффициентов;
- проверка фактической точности траектории;
- проверка реального бокового зазора при обходе;
- проверка, хватает ли текущего простого планировщика для всех конфигураций препятствий.

## API и веб-сокеты L3

`L3` — это основной внешний уровень нового автономного контура.

Именно через него фронтенд должен:
- задавать цель одной точкой;
- задавать маршрут;
- передавать список заранее известных препятствий;
- запускать автономное движение;
- отменять выполнение;
- получать текущее состояние маршрута и статусы.

### REST API L3

#### Задать цель одной точкой

```text
POST /api/v1/l3/goal
```

Тело:

```json
{
  "target": {
    "x_cm": 120.0,
    "y_cm": 30.0
  },
  "known_obstacles": [
    {
      "x_cm": 70.0,
      "y_cm": 20.0,
      "radius_cm": 8.0
    }
  ]
}
```

Ответ:

```json
{
  "status": "tracking",
  "mode": "point",
  "planner_status": "planned",
  "target_x_cm": 59.0,
  "target_y_cm": 33.0,
  "active_point_index": 0,
  "total_points": 3,
  "distance_error_cm": null,
  "heading_error_deg": null,
  "target_heading_deg": null,
  "linear_speed_cm_per_sec": 0.0,
  "angular_speed_deg_per_sec": 0.0
}
```

Важно:
- если планировщик вставил обходные точки, `target_x_cm`, `target_y_cm` в ответе — это текущая активная точка спланированного маршрута, а не обязательно исходная пользовательская цель.

#### Задать маршрут

```text
POST /api/v1/l3/route
```

Тело:

```json
{
  "points": [
    {
      "x_cm": 100.0,
      "y_cm": 0.0
    },
    {
      "x_cm": 140.0,
      "y_cm": 20.0
    }
  ],
  "known_obstacles": [
    {
      "x_cm": 60.0,
      "y_cm": 0.0,
      "radius_cm": 8.0
    }
  ]
}
```

#### Выполнить один шаг управления

Если управление верхним уровнем будет идти дискретными шагами:

```text
POST /api/v1/l3/step
```

Ответ:

```json
{
  "status": "tracking",
  "mode": "route",
  "planner_status": "planned",
  "target_x_cm": 71.0,
  "target_y_cm": 13.0,
  "active_point_index": 0,
  "total_points": 4,
  "distance_error_cm": 22.4,
  "heading_error_deg": 15.0,
  "target_heading_deg": 18.0,
  "linear_speed_cm_per_sec": 11.0,
  "angular_speed_deg_per_sec": 30.0
}
```

#### Получить текущее состояние `L3`

```text
GET /api/v1/l3/state
```

#### Отменить текущую цель или маршрут

```text
POST /api/v1/l3/cancel
```

#### Сбросить маршрут и состояние перед новым запуском

Если понадобится отдельная команда инициализации:

```text
POST /api/v1/l3/reset
```


```json
{
  "type": "l3_state",
```

Допустимые статусы:
- `idle`

```json
  "total_points": 5
}
```

### Что калибровать в L3

Для `L3` калибруется не железо, а поведение верхнего уровня.

Нужно подобрать:
- `l3_position_tolerance_cm`
- `l3_linear_speed_gain`
- `l3_angular_speed_gain`
- `l3_max_linear_speed_cm_per_sec`
- `l3_max_angular_speed_deg_per_sec`
- `l3_obstacle_stop_distance_cm`
- `l3_obstacle_slowdown_distance_cm`
- `l3_planner_obstacle_clearance_cm`
- `l3_planner_max_detour_offset_cm`

Что влияет на цели команды:
- `l3_obstacle_stop_distance_cm`
  влияет на выполнение требования остановки не ближе 20 см;
- `l3_obstacle_slowdown_distance_cm`
  влияет на плавность торможения перед препятствием;
- `l3_planner_obstacle_clearance_cm`
  влияет на минимальный зазор при объезде;
- `l3_planner_max_detour_offset_cm`
  влияет на возможность уложиться в допустимый коридор;
- `l3_position_tolerance_cm`
  влияет на выполнение требования по точности достижения точки и траектории.
