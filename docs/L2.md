# Уровень L2

## Назначение

`L2` — это математический уровень движения машинки.

Именно на этом уровне:
- желаемое движение корпуса переводится в команды левого и правого борта;
- измерения, пришедшие снизу, переводятся в оценку текущего состояния машинки;
- хранится текущее состояние нового контура движения.

`L2` не управляет устройствами напрямую и не принимает навигационных решений. Он только принимает данные, выполняет расчёты и возвращает результат.

## Что входит в L2

В текущей реализации в `L2` входят:
- модели данных уровня;
- кинематика корпуса и бортов;
- оценка состояния машинки;
- контроллер желаемой скорости корпуса;
- единый сервис уровня.

## Какие величины используются в L2

Для расчётов в уровне используются следующие величины:
- `v` — линейная скорость корпуса;
- `omega` — угловая скорость корпуса;
- `v_L`, `v_R` — линейные скорости левого и правого борта;
- `u_L`, `u_R` — команды левого и правого борта в процентах;
- `x`, `y` — координаты машинки на плоскости;
- `theta` — угол поворота машинки;
- `a_long` — продольное ускорение;
- `dt` — шаг времени;
- `l` — расстояние между центрами бортов.

В текущем коде используются такие единицы:
- `v` — `см/с`;
- `omega` — `град/с` на внешнем интерфейсе и `рад/с` внутри формул кинематики;
- `u_L`, `u_R` — проценты в диапазоне `[-100; 100]`;
- `x`, `y` — сантиметры;
- `theta` — градусы;
- `a_long` — `м/с^2`;
- `l` — сантиметры.

## Что делает L2

`L2` выполняет следующие действия:
- принимает желаемую линейную и угловую скорость корпуса;
- рассчитывает команды левого и правого борта;
- ограничивает команды допустимым диапазоном;
- принимает измерения снизу;
- обновляет положение, угол и скорости;
- хранит текущее состояние машинки;
- возвращает единый снимок состояния уровня.

## Полная цепочка расчёта в L2

На текущий момент внутри уровня выполняется следующая последовательность.

### 1. Приходят желаемые скорости корпуса

На вход `L2` подаются:
- `linear_speed_cm_per_sec`
- `angular_speed_deg_per_sec`

Это желаемое движение корпуса машинки.

### 2. Скорости корпуса переводятся в скорости бортов

Используются формулы дифференциальной машинки:

```text
v_R = v + omega * l / 2
v_L = v - omega * l / 2
```

где:
- `v` — линейная скорость корпуса;
- `omega` — угловая скорость корпуса;
- `l` — база между бортами;
- `v_R` — скорость правого борта;
- `v_L` — скорость левого борта.

Так как в коде `omega` приходит в `град/с`, перед подстановкой в формулы она переводится в `рад/с`.

### 3. Скорости бортов переводятся в команды бортов

После этого рассчитываются команды:

```text
u_R = v_R / v_R_max * 100
u_L = v_L / v_L_max * 100
```

где:
- `u_R`, `u_L` — команды бортов в процентах;
- `v_R_max`, `v_L_max` — максимальные скорости бортов при команде `100%`.

### 4. Команды ограничиваются диапазоном

Команды не должны выходить за диапазон `[-100; 100]`.

Для этого сначала вычисляется:

```text
m = max(abs(u_L), abs(u_R))
```

Если `m <= 100`, команды оставляются как есть.

Если `m > 100`, обе команды уменьшаются пропорционально:

```text
u_L = u_L * 100 / m
u_R = u_R * 100 / m
```

Это позволяет сохранить соотношение между левым и правым бортом.

### 5. Команды передаются вниз

На выходе этого расчёта `L2` выдаёт:
- `left_percent`
- `right_percent`

Это уже готовые команды для нижнего уровня.

### 6. Измерения снизу используются для обновления состояния

После прихода измерений `L2` использует:
- `angular_speed_z_deg_per_sec`
- `accel_x_m_s2`
- `accel_y_m_s2`
- `accel_z_m_s2`
- `distance_cm`

В текущей реализации для оценки состояния реально используются:
- угловая скорость вокруг вертикальной оси;
- продольное ускорение;
- расстояние до препятствия сохраняется в состоянии как измеренное значение.

### 7. Обновляется линейная скорость

Если используется продольное ускорение, линейная скорость меняется так:

```text
v(t + dt) = v(t) + a_long * dt
```

Так как в коде скорость хранится в `см/с`, а ускорение приходит в `м/с^2`, применяется перевод:

```text
v_cm_s(t + dt) = v_cm_s(t) + 100 * a_long * dt
```

### 8. Обновляется угол

Угол поворота считается по измеренной угловой скорости:

```text
theta_next = theta + omega_z * dt
```

То есть в новом контуре угол строится именно в `L2`, а не приходит готовым снизу.

### 9. Обновляются координаты

Положение машинки на плоскости считается по кинематике:

```text
x_dot = v * cos(theta)
y_dot = v * sin(theta)
theta_dot = omega
```

В дискретном виде:

```text
x_next = x + v * cos(theta) * dt
y_next = y + v * sin(theta) * dt
theta_next = theta + omega * dt
```

В текущей реализации направление движения за шаг считается по среднему углу на этом шаге.

## Обратное преобразование бортов в движение корпуса

Если известны команды бортов, `L2` умеет восстановить оценку движения корпуса.

Сначала восстанавливаются скорости бортов:

```text
v_L = u_L / 100 * v_L_max
v_R = u_R / 100 * v_R_max
```

Потом восстанавливается движение корпуса:

```text
v = (v_R + v_L) / 2
omega = (v_R - v_L) / l
```

Эти формулы используются для обратного пересчёта в `src/application/services/kinematics.py`.

- `right_track_max_speed_cm_per_sec` — скорость правого борта при команде `100%`.

## Что обязательно нужно откалибровать

Для реальной точности работы `L2` обязательно нужно подобрать:
- `track_width_cm`
- `left_track_max_speed_cm_per_sec`
- `right_track_max_speed_cm_per_sec`

### Что означает каждый параметр

`track_width_cm` влияет на расчёт поворота:
- если значение слишком маленькое, поворот будет рассчитываться слишком резким;
- если значение слишком большое, поворот будет рассчитываться слишком вялым.

`left_track_max_speed_cm_per_sec` и `right_track_max_speed_cm_per_sec` влияют на связь между:
- желаемой скоростью корпуса;
- командами бортов;
- обратным восстановлением движения корпуса по командам бортов.

Если эти значения выбраны неверно:
- машинка будет ехать не с той линейной скоростью, которая ожидается в модели;
- поворот будет считаться с ошибкой;
- состояние `x`, `y`, `theta`, `v`, `omega` будет постепенно расходиться с реальностью.

### Что сейчас используется

Сейчас в коде стоят стартовые ориентиры:
- `track_width_cm = 17.0`
- `left_track_max_speed_cm_per_sec = 30.0`
- `right_track_max_speed_cm_per_sec = 30.0`

Это не измеренные значения вашей машинки, а временные исходные коэффициенты.

## Что L2 делает сам

`L2` сам выполняет:
- пересчёт движения корпуса в команды бортов;
- обратный пересчёт команд бортов в движение корпуса;
- ограничение команд диапазоном `[-100; 100]`;
- обновление угла по угловой скорости;
- обновление линейной скорости по продольному ускорению;
- обновление координат на плоскости;
- хранение текущего состояния.

## Что L2 не делает

`L2` не делает:
- работу с GPIO, I2C, PWM и драйверами;
- прямое чтение устройств;
- выбор точки назначения;
- построение маршрута;
- обход препятствий;
- принятие решения о следующем действии.

## Текущие упрощения уровня

На текущий момент `L2` реализован в упрощённом рабочем виде.

Это значит:
- угол считается по угловой скорости без отдельной фильтрации;
- линейная скорость может обновляться прямым интегрированием продольного ускорения;
- нет сложной фильтрации уровня EKF;
- нет слияния нескольких независимых источников скорости;
- нет автоматической подстройки коэффициентов кинематики.

То есть уровень завершён как программная реализация, но для высокой точности ему ещё нужна реальная калибровка параметров.

## API и веб-сокеты L2

`L2` — это первый уровень нового контура, который имеет смысл отдавать наружу как стабильный API для ручного управления и состояния.

Через `L2` фронтенд может:
- задавать желаемую линейную и угловую скорость;
- сбрасывать оценку состояния;
- получать текущее состояние машинки;
- подписываться на поток состояния.

### REST API L2

#### Задать желаемую скорость корпуса

```text
POST /api/v1/l2/cmd-vel
```

Тело:

```json
{
  "linear_speed_cm_per_sec": 12.0,
  "angular_speed_deg_per_sec": 30.0
}
```

Ответ:

```json
{
  "x_cm": 0.0,
  "y_cm": 0.0,
  "heading_deg": 0.0,
  "linear_speed_cm_per_sec": 12.0,
  "angular_speed_deg_per_sec": 30.0,
  "left_percent": 8.5,
  "right_percent": 15.5,
  "distance_cm": 42.0
}
```

#### Остановить контур L2

```text
POST /api/v1/l2/stop
```

Ответ:

```json
{
  "x_cm": 0.0,
  "y_cm": 0.0,
  "heading_deg": 0.0,
  "linear_speed_cm_per_sec": 0.0,
  "angular_speed_deg_per_sec": 0.0,
  "left_percent": 0.0,
  "right_percent": 0.0,
  "distance_cm": 42.0
}
```

#### Сбросить оценку состояния

```text
POST /api/v1/l2/reset-state
```

Тело:

```json
{
  "x_cm": 0.0,
  "y_cm": 0.0,
  "heading_deg": 0.0,
  "linear_speed_cm_per_sec": 0.0,
  "angular_speed_deg_per_sec": 0.0
}
```

#### Получить текущее состояние

```text
GET /api/v1/l2/state
```

Ответ:

```json
{
  "x_cm": 15.2,
  "y_cm": 3.1,
  "heading_deg": 12.0,
  "linear_speed_cm_per_sec": 10.0,
  "angular_speed_deg_per_sec": 20.0,
  "left_percent": 7.0,
  "right_percent": 13.0,
  "distance_cm": 37.0
}
```

### Что калибровать в L2

Для `L2` калибровка обязательна.

Нужно подобрать:
- `track_width_cm`
- `left_track_max_speed_cm_per_sec`
- `right_track_max_speed_cm_per_sec`

Желательно дополнительно проверить:
- знак `angular_speed_z_deg_per_sec`;
- какая ось акселерометра является продольной;
- насколько быстро накапливается ошибка `x`, `y`, `heading` на прямом заезде и на повороте.
